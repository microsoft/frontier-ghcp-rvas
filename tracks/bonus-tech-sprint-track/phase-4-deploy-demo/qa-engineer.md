# Phase 4: Ship and Demo -- QA Engineer Tasks

[Back to Phase 4 Overview](../phase-4-deploy-demo.md) | [Back to Technical Team Sprint Track](../../bonus-tech-sprint-track.md)

**Time: ~1 hour tasks + demo/retro**

## Tasks

1. **Final regression run** -- Run the full E2E test suite against the production deployment.

2. **Smoke test on production** -- Manually verify the critical path: browse trails, view a trail detail page, submit a condition report. Confirm the deployed version works like local.

3. **Generate the final test report** -- Total tests, pass rate, and a coverage summary. Have this ready for the demo.

4. **Document known issues** -- Create or update `docs/known-issues.md` listing bugs that the team will not fix before the demo. Include severity and workarounds.

5. **Demo contribution** -- Show the test suite, test results, and any interesting bugs you caught during the day.

## Verification

- [ ] Final test suite run completed
- [ ] Production smoke test passed
- [ ] Test report ready for demo
- [ ] Known issues documented
