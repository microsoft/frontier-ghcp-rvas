# User Story Template

Use this structure when creating GitHub Issues for user stories.

---

**Title format:** `[Epic] As a <role>, I want <goal>`

**Body:**

```markdown
## User Story

**As a** [type of user],
**I want** [goal],
**so that** [reason].

## Acceptance Criteria

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Priority

Must / Should / Could / Won't

## Size

S / M / L

## Epic

Trails / Conditions / Dashboard / Accounts

## Notes

Any additional context, links, or design references.
```

**Labels to apply:**

- Epic: `epic:trails`, `epic:conditions`, `epic:dashboard`, `epic:accounts`
- Priority: `priority:must`, `priority:should`, `priority:could`
- Size: `size:S`, `size:M`, `size:L`
- Type: `user-story`
